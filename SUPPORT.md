# Support

## How to get help

1. **Documentation** — start with the [docs index](docs/index.md), the [quick start](docs/getting-started/quick-start.md), and the [FAQ](docs/troubleshooting/faq.md).
2. **Troubleshooting guides** — [docs/troubleshooting/](docs/troubleshooting/) covers common errors and diagnostics.
3. **GitHub Discussions** — usage questions and design discussions.
4. **GitHub Issues** — bugs (with minimal repro) and feature requests via the issue templates.

## What to include in a bug report

- Package name(s) and version(s), .NET SDK/runtime version, target framework
- Minimal reproduction (a failing test or small snippet)
- Expected vs actual behavior; full exception output where relevant

## Response expectations

This is an open-source project maintained by Koras Technologies. Issues are triaged weekly; security reports follow the timeline in [SECURITY.md](SECURITY.md). No commercial support contracts are currently offered.
